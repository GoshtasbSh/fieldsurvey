# FieldSurvey Python Sidecar

A FastAPI app deployed as a single Vercel Fluid Compute Python 3.13 function. It
hosts the 4 statistically-heavy M7 wave-1 analyses cards that are not
expressible as a single Supabase RPC:

| Card | Endpoint                            | Method | Notes                          |
| ---- | ----------------------------------- | ------ | ------------------------------ |
| A21  | `POST /sidecar/compute/A21_finish`  | Monte Carlo bootstrap on daily counts |
| A25  | `POST /sidecar/compute/A25_velocity`| PELT change-point detection (ruptures)|
| A11  | `POST /sidecar/compute/A11_kde`     | Gaussian FFT KDE (KDEpy)              |
| A8   | `POST /sidecar/compute/A8_gi_star`  | Getis-Ord Gi\* with KNN + 999 perms   |

Each handler writes its result back into the shared `dashboard_cache` table so
the Next.js dispatcher can return the cached payload on subsequent reads for up
to 15 minutes (matches the M6 cache TTL).

## Surfaces

- `GET /sidecar/healthz` → `{ "ok": true }`
- `GET /sidecar/version` → `{ "version": "1.0.0" }`
- Four `POST /sidecar/compute/<card_id>` endpoints (above)

## Environment

| Variable                     | Where                              |
| ---------------------------- | ---------------------------------- |
| `SUPABASE_URL`               | Vercel project env (read at boot)  |
| `SUPABASE_SERVICE_ROLE_KEY`  | Vercel project env (server-only)   |
| `NEXT_PUBLIC_SIDECAR_URL`    | Set on Next.js side so the dispatcher knows where to POST |

## Deploy

The sidecar deploys as part of the main Vercel project — `vercel.json`
(generated by Batch 6) declares:

```json
{
  "functions": {
    "sidecar/app.py": { "runtime": "python3.13", "maxDuration": 60, "memory": 1024 }
  },
  "rewrites": [
    { "source": "/sidecar/:path*", "destination": "/sidecar/app.py" }
  ]
}
```

After `vercel --prod`, the function shows up in the Vercel dashboard with the
shared `/sidecar/*` route prefix.

## Local development

```bash
cd sidecar
python3.13 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# Run locally with uvicorn:
SUPABASE_URL=... SUPABASE_SERVICE_ROLE_KEY=... \
  uvicorn sidecar.app:app --reload --port 8001
```

Then hit:

```bash
curl http://localhost:8001/sidecar/healthz
```

## Smoke tests

```bash

# A21 Monte Carlo

  -H 'content-type: application/json' \
  -d '{"project_id":"<uuid>","history":[5,7,4,8,5,6,5,5,7,4],"target":100,"start":"2026-06-01"}'

curl -X POST https://<deployment>/sidecar/compute/A25_velocity \
  -H 'content-type: application/json' \
  -d '{"project_id":"<uuid>","daily_counts":[3,3,3,3,3,3,3,3,3,3,9,9,9,9,9,9,9,9,9,9]}'

# A11 KDE
curl -X POST https://<deployment>/sidecar/compute/A11_kde \
  -H 'content-type: application/json' \
  -d '{"project_id":"<uuid>","points":[[-82.71,29.13],[-82.71,29.13]],"bandwidth":0.005,"grid_size":64}'

```

## Tests

```bash
cd <repo root>
python3.13 -m pytest sidecar/tests/ -v
```

Each card has a unit test that exercises `compute()` directly without hitting
Supabase, so tests run offline.
